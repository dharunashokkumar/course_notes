Lecture 01 Complete Explanation

Card 1
What is a protocol in computer networking?

Protocol is a set of rules that governs how devices communicate over a network. It defines the format, timing, sequencing, and error handling required for successful data exchange. Protocols enable functionalities such as file transfer, email, and web browsing by providing a common language for heterogeneous systems.

Card 2
How is network architecture visualized?

Network architecture is a structured view of how two remote computers communicate, combining a protocol stack with the underlying physical technology (wired, wireless, etc.). It shows the layers of rules that data passes through and the devices that implement each layer. This abstraction allows different physical setups to interoperate using the same standard protocols.

Card 3
What are the main responsibilities of the Physical Layer (Layer 1)?

The Physical Layer handles signal transmission between devices. It converts digital data ↔ analog signals for transmission over media such as copper cables, fiber optics, or radio waves. Its sole purpose is to move bits from one endpoint to another; it does not interpret the data content.

Card 4
What is a hub (repeater) and what are its key properties?

Hub (or Repeater) is a Layer 1 device that simply amplifies and forwards electrical signals.

  • Acts as a multi‑port concentrator for all connected hosts.
  • Regenerates weakened signals, but makes no intelligent forwarding decisions.
  • All ports share the same collision domain and same broadcast domain.
Card 5
Define collision domain and broadcast domain.

Collision Domain: The network segment where simultaneous transmissions can interfere, causing collisions that require retransmission and waste bandwidth. Broadcast Domain: The set of devices that receive each other's broadcast frames; a broadcast sent by one host is heard by all others in the same domain. Devices can share a broadcast domain while belonging to separate collision domains.

Card 6
How does the Data Link Layer (Layer 2) improve connectivity?

The Data Link Layer introduces Layer 2 switches that divide collision domains, reducing the chance of collisions and improving effective bandwidth. However, all ports remain in the same broadcast domain, so broadcasts still reach every device on the LAN. This layer adds MAC addressing and frame error detection.

Card 7
What is the role of the Network Layer (Layer 3) and a router?

The Network Layer connects different networks (different broadcast domains) by routing packets based on logical addresses (IP). A router (or Layer 3 switch) divides both collision and broadcast domains and selects the optimal path for each packet across multiple networks. It operates up to the Network Layer, treating higher layers as payload.

Card 8
What services does the Transport Layer (Layer 4) provide?

The Transport Layer offers process‑to‑process communication between applications on different hosts. It adds error control, flow control, and traffic management (e.g., congestion avoidance). Common protocols are TCP (reliable, connection‑oriented) and UDP (unreliable, connection‑less).

Card 9
List the five layers of the TCP/IP protocol stack and their primary functions.

TCP/IP Stack (top‑down):

  1. Application – User‑level services (HTTP, FTP, SMTP).
  2. Transport – Process‑to‑process delivery, error/flow control (TCP, UDP).
  3. Network – Routing between networks (IPv4, IPv6).
  4. Data Link – Local framing, MAC addressing, collision handling (Ethernet, Wi‑Fi).
  5. Physical – Bit‑level signal transmission over media.
Card 10
How do intermediate devices handle packets and what are examples of cross‑layer protocols?

Each intermediate device opens a packet only up to its own layer:

  • Hub – reads up to Physical; everything else is payload.
  • Layer 2 Switch – reads up to Data Link; higher layers are payload.
  • Router – reads up to Network; transport and above remain payload. At source and destination, the packet is processed through all layers. Cross‑layer protocols span two layers, e.g., DNS (Application ↔ Transport), ARP (Network ↔ Data Link), SNMP (Transport ↔ Network), and DHCP (multiple layers).
